CmdFreak 1.3.0

1. CmdFreak summary

Several information systems and application software suites use a RDBMS (Relational Database Management System) as a persistence mechanism for their own data. The use of RDBMSs continues to expand. Most RDBMSs are used as a background service for information systems and application software. However, such systems can also be accessed directly for data mining and faster business operations and maintenance. An appropriate RDBMS can be chosen to match the characteristics of the target information system and application software. As a result, multiple RDBMSs are often used simultaneously in the same business operation and maintenance environment. However, operating multiple RDBMSs introduces the following issues:
  • Different data reference software applications need to be provided, which may increase the learning cost.
  • Data reference software needs to be installed on each client-side PC. The cost of introduction is therefore higher with a greater number of client PCs.
CmdFreak is a relational database viewer (data reference software) that provides solutions for the above problems.
  • CmdFreak supports the major free RDBMS MySQL, PostgreSQL and MariaDB systems, providing a common user interface for these RDBMSs and enabling data reference as a common operation. This eliminates the need to learn different database viewer tools for multiple RDBMSs, which helps to reduce related learning costs.
  • CmdFreak works as a web application, and comes with an all-in-one installer incorporating web server software. This eliminates the need for web server pre-configuration and the use of script language. As a result, the client-side installation cost is practically zero.
CmdFreak provides the following functions for database reference:
  • List records/paging
  • Table selection
  • Table information reference
  • Column filtering and sorting
  • DBMS selection

  • Memory:1 GB or more
  • Free disk space: 20 MB or more
  • Operating systems/editions supported:

    MySQL 8.4

4. Installation

For Windows
Execute cmdfreak-1.3.0.msi and launch the installer.
Follow the on-screen dialog prompts.
The installer prompts the user to enter the following information:
  • Listening port number of Web server
    The port number that Web browser connects to.
  • Listening port number of CmdFreak service
    The port number of REST-API that Web server internally connects to.
CmdFreak installer performs the following steps during the installation.
  • Register exceptions of Windows firewall for CmdFreak.
  • Register service of CmdFreak to the target system.
  • Auto start of CmdFreak service and Web server.
An overwrite installation (Repair) can be performed by re-running cmdfreak-1.3.0.msi after CmdFreak installation. Note that the items shown below for overwrite installation.
  • Stop CmdFreak service before overwrite installation. If you perform the overwrite installation without service stop, restart of operating system is required after overwrite installation done.
  • During overwrite installation, port numbers which Web server and CmdFreak service use might be changed into default. Confirm the port number after overwrite installation.
For Linux
Install cmdfreak-1.3-0.el9.x86_64.rpm or cmdfreak-1.3-0.el10.x86_64.rpm using following command.
# rpm -ivh cmdfreak-1.3-0.el9.x86_64.rpm
or
# rpm -ivh cmdfreak-1.3-0.el10.x86_64.rpm
Note that nginx 1.20 or later version, and unixODBC need to be installed before CmdFreak installation.
CmdFreak installer performs the following steps during the installation.
  • Change httpd_can_network_connect of SELinux into on.
  • Register exceptions of firewall for CmdFreak.
  • Register service of CmdFreak to the target system.
  • Auto start of CmdFreak and restart of Web server(nginx).
If an existing nginx instance is already in operation, restarting nginx may affect active sites.

5. Upgrade installation

In order to upgrade from CmdFreak 1.2.0, perform installation according to following steps. Linux edition is offered from 1.3.0 hence upgrade installation is out of support.
  1. Stop CmdFreak service.
  2. Copy <Installation folder>\sample.dat file to other folder.
  3. Uninstall CmdFreak 1.2.0.
  4. Install CmdFreak 1.3.0.
  5. Stop CmdFreak service.
  6. Rename, copy and overwrite to <Installation folder>\cmdfreak.dat file from sample.dat which is copied in previous step.
  7. Modify <Installation folder>\conf\nginx.conf and <Installation folder>\cmdfreak.conf if necessary.
  8. Start CmdFreak service.

6. Uninstallation

For Windows
Select Remove CmdFreak with execution of cmdfreak-1.3.0.msi or perform uninstallation of CmdFreak from Control Panel.
CmdFreak uninstallation performs following things during uninstallation.
  • Auto stop of CmdFreak service and Web server.
  • Deletion of CmdFreak service and Web server from the target system.
  • Deletion of Windows firewall exception for CmdFreak.
For Linux
Uninstall the CmdFreak using following command.
# rpm -e cmdfreak
CmdFreak uninstaller performs the following steps during the uninstallation.
  • Auto stop of CmdFreak service and Web server(nginx).
  • Unregister exceptions of firewall for CmdFreak.
  • Remove the service of CmdFreak from the target system.
  • Starts Web server
If an existing nginx instance is already in operation, restarting nginx may affect active sites.
During uninstallation, the SELinux configuration (httpd_can_network_connect) is not reverted to pre-installation status. Revert it manually if necessary.

7. Login to CmdFreak

Connect to the URL shown below by Web browser for use of CmdFreak after installation. If main console is displayed after login sequence of CmdFreak, it shows the installation is successfully completed.

http://<Host name or IP address of CmdFreak>:<Port number of Web server>

In case of specification of default installations parameters : http://localhost:7120

Initial login user name and password are follows.

User name : admin
Password : manager

If main console is not appeared or an error occurred, confirm the things shown below.
  • When host name is specified in the URL for the connection to CmdFreak from browser, confirm that the host name can be resolved. In case the host name is not resolved, change the host name to another name that can be resolved, and then retry.
  • Confirm that CmdFreak(nginx.exe) is registered as an exception of Windows firewall. If not, register nginx.exe to the Windows firewall as an exception.
  • Confirm that the specified port number is not already in use by another process. If it is, choose a different port number as described below.

8. Start/Stop CmdFreak service

For Windows
Start and stop of CmdFreak service can be performed by Windows start menu.
  • [CmdFreak]-[Start CmdFreak service] : Start service
  • [CmdFreak]-[Stop CmdFreak service] : Stop service
For Linux
Start and stop operation for CmdFreak service can be performed by following command.
Start: # systemctl start cmdfreak
Stop: # systemctl stop cmdfreak

9. Change IP address/host name and port number

CmdFreak uses two kinds of IP addresses/host names and ports.
  • IP address/host name and port of Web server
  • IP address/host name and port of CmdFreak service
In order to change the IP address/host name and port number after installation, following steps are required.
  1. Stop CmdFreak service.
  2. Change IP address/host name and port number.

    In case you change IP address/host name and port number which CmdFreak service uses:
    Specify IP address/host name and port number to servicehost and serviceport properties in <Installation folder>\cmdfreak.conf
    Specify IP address/host name and port number to proxy_pass of location /api/ section in <Installation folder>\conf\nginx.conf

    In case you change IP address/host name and port number which Web server uses:
    Change listen and server_name in <Installation folder>\conf\nginx.conf

  3. Change firewall setting if necessary.
  4. Start CmdFreak service.

10. Connection using SSL/TLS

In order to connect to CmdFreak using SSL/TLS from browser, following steps are required.
  1. Stop CmdFreak service.
  2. Add SSL/TLS settings to nginx.conf.
    Add followings to server directive in <Installation folder>\conf\nginx.conf (for Windows) or /etc/nginx/conf.d/cmdfreak_nginx.conf (for Linux)
    (Regarding certificate and private key, modify the path according to your environment.)
    ssl on;
    ssl_certificate "D:\\work\\server.crt";
    ssl_certificate_key "D:\\work\\server.key";
  3. Change listen port number into 443 or appropriate HTTPS port number.
  4. Start CmdFreak service.

11. About ODBC driver

CmdFreak uses ODBC for connecting to RDBMS. Therefore ODBC driver needs to be installed into same server where CmdFreak works. ODBC driver needs to be chosen depending on the type of target RDBMS. For example, If you want to use MySQL, ODBC driver for MySQL is required. CmdFreak works as a 64-bit application hence ODBC driver has to be 64-bit edition of driver. ODBC driver is provided by the vendor who provides RDBMS. Download it from the vendor Web site.

12. Management limitations

  • Maximum length of table name: 64 (Depends on the connected DBMS. PostgreSQL = 63 bytes, MySQL/MariaDB = 64 characters)
  • Maximum length of column name: 64 (Depends on the connected DBMS. PostgreSQL = 63 bytes, MySQL/MariaDB = 64 characters)
  • Maximum length of column value(characters): 63 (64th or later characters are not displayed)
  • Maximum number of tables: 300 (301st or later tables are not displayed)
  • Maximum number of columns: 50 (51st or later columns are not displayed)
